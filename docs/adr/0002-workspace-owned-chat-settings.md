# Make Workspaces Own Live Chat Settings

We will require every **Conversation** to belong to a **Workspace**, make each **Workspace** own the live chat configuration used for new **Turns**, and keep selectable models in an internal **Model Catalog** table rather than a frontend constant list. This means **Conversation** stays context-only instead of snapshotting model or system-prompt settings, each new **Turn** reads the workspace's current **Selected Model** and **System Message**, title generation remains separate from workspace model choice, and workspaces are non-deletable but archivable and restorable. We chose this over standalone conversations and per-conversation setting snapshots because it gives users an explicit place to manage model behavior, keeps the data model simpler for the current scope, and centralizes dynamic model-specific settings without introducing user-facing model administration.
