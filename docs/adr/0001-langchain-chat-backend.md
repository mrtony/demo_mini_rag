# Use LangChain Runnable Streaming for the Chat Backend

We will replace the backend's direct OpenAI Python SDK integration with `LangChain + langchain-openai`, while keeping the app's existing SSE contract (`conversation.created`, `conversation.title`, `message.created`, `message.delta`, `message.done`, `error`) stable. The backend will use runnable-based streaming with `astream_events()`, map LangChain events into an internal app-owned chat event envelope, rename `OpenAIService` to a neutral `ChatService`, and keep route-level control over stop/disconnect, incremental DB writes, and temporary-title-then-final-title behavior. We are explicitly not introducing LangGraph in this migration because the current roadmap is still a focused chat flow rather than a graph-shaped workflow, and keeping LangChain behind an internal event adapter preserves a cleaner path for future RAG, tools, or LangGraph adoption without forcing the frontend or routes to depend on framework-native event semantics.
